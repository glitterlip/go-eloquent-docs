Relations

Currently,we have belongsto,belongstomany,hasmany,hasone,morphbymany,morphone,morphmany,morphto
,morphtomany relations.

When define relation method,parameter with prefix self means current model database field , related means related
model database field.

How did it work?

Basiclly process relations only need 5 steps. Let’s take youtube as an example.
So we have a user table and a video table, and user table id column correspond to video table user_id column.
We want get first ten users with their videos.
Let’s do this step by step.

  1. First of all ,we get the self(user) records.select * from users offset 0 limit 10
  2. Secondly we extract all the users’ id as ids
  3. Next we get all the related records(videos) which user_id column is in the users’
    ids select * from videos where user_id in (ids)
  4. Then we need group the videos by videos’ user_id column so we have a map[string][]video map,map key is videos’
    user_id, value is videos that have the same user_id
  5. Finally, we iterate over all the users,check if there is a group of videos in the map that the key equals to the
    user’s id. If there is one ,we set the group of videos to the user videos filed

How to use

For a model , all relations are a filed of the model struct.
For a User model, it is a struct with a type []Video field Videos.
For a Video model, it is a struct with a type User field User. Further more we need more details to process the
relations.

A model must have a field of pointer of goeloquent.EloquentModel so that we can use model features.
And we also need to know model’s primary key column.

Add a tag to the model struct filedgoelo:"RelationType:RelationMethod"
RelationType can be HasOne,BelongsTo,HasMany,BelongsToMany,MorphOne,MorphMany,MorphTo,MorphToMany
,MorphByMany
RelationMethod is the model method defines the relation that returns a pointer of RelationBuilder
